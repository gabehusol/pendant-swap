# Screenshots for the README

Capture these four images and save them in this folder with the exact names.
The main README already references them, so they appear automatically once added.

Open the app first at `http://127.0.0.1:8000/web/index.html`.

| File name | What to capture |
|---|---|
| `01-overview.png` | The whole app screen right after opening, before filling anything in. |
| `02-generate.png` | Generate mode selected, so the AI settings area is showing (key field, model, switches). |
| `03-results.png` | The Results panel after a run: the result image on the left, the PASS or FAIL badge and attempts list on the right. |
| `comparison.png` | A side by side of a plain Gemini result (left) and a Pendant Swap result (right). Make this in any image editor by placing the two pictures next to each other. |

For `comparison.png`, generate the "plain Gemini" half in Google AI Studio: upload the
model photo, add the pendant photo, and ask it to put the pendant on the necklace. That
shows the oversize and detail drift that this tool fixes.
